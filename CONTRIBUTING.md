# Contributing to InterviewLab

Thanks for helping improve InterviewLab.

## Before opening a change

- Search existing issues and pull requests for related work.
- Open an issue first for substantial product, data-model, or dependency changes.
- Keep interview feedback discussion-based and avoid exposing hidden solution
  patterns before a candidate submits.
- Do not add copyrighted problem statements or solution text without permission.
  Prefer links, identifiers, and original summaries.

## Development workflow

1. Fork the repository and create a focused branch.
2. Install Node.js 22.13 or newer and run `npm ci`.
3. Make the smallest coherent change.
4. Run `npm test` and `npm run lint`.
5. Open a pull request explaining the behavior and validation performed.

Never commit API keys, candidate data, resumes, or interview transcripts.
